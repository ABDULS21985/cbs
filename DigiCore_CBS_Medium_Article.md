## What's Actually in the Box

Let's skip the slide deck and talk about what's been built.

### A Real General Ledger — Not a Transaction Log

Most "core banking" startups outsource the ledger or fake it with a transaction table. DigiCore ships a proper double-entry general ledger with:

- **Chart of Accounts management** with hierarchical account structures
- **Journal entry posting** with batch processing and multi-line support
- **Subledger reconciliation** with automated break detection
- **GL balance reporting** across periods and currencies
- **Funds Transfer Pricing (FTP)** for margin allocation across business lines

If your core banking system can't produce a trial balance from its own data, it's not a core banking system. Ours can.

### Customer 360 — Built for Regulated Banking

This isn't a CRM contact card. It's a full Customer Information File (CIF) purpose-built for banking:

- **Multi-address, multi-contact, multi-identification** support with document verification
- **Customer lifecycle management** — PROSPECT to ACTIVE to DORMANT to CLOSED, with every state transition logged
- **KYC workflows** with BVN verification, document upload, review panels, and decision trails
- **Customer relationships** — spouse, guarantor, beneficial owner, corporate hierarchy
- **Risk rating assignment** with behavioral scoring
- **Onboarding wizard** — a guided, multi-step digital onboarding journey with draft saving and resume
- **Dynamic segmentation** — rule-based customer grouping with operators (EQ, GT, LT, IN, RANGE) and automatic, manual, or scheduled assignment

Every customer interaction, every account, every loan, every card, every transaction — visible from a single 360-degree view with tabbed navigation across twelve dimensions.

### Full-Spectrum Lending — Not Just Personal Loans

DigiCore handles the complete lending lifecycle:

- **Loan origination** from application through credit assessment, approval, and disbursement
- **Flexible repayment schedules** — amortizing, bullet, balloon, custom — generated by a real schedule engine
- **Collateral management** with valuation tracking, revaluation history, and multiple collateral types
- **Loan restructuring** with modification logging and audit trails
- **Mortgage processing** with property-linked collateral workflows
- **Microfinance** support with group lending models
- **POS lending** for point-of-sale financing
- **Collections workflows** with campaign automation and recovery tracking

Plus dedicated credit risk dashboards, facility management, and integration with the ECL engine for provisioning.

### Deposits That Actually Work Like a Bank

- **Fixed deposits** with maturity actions (reinvest, liquidate, transfer), penalty management, and interest calculation
- **Recurring deposits** with installment tracking, missed payment handling, and frequency configuration
- **Savings goals** with target tracking, contribution monitoring, and analytics
- **Escrow accounts** for regulated holding and release workflows
- **Interest posting** with tiered rates, overrides, and posting history

This is where integration nightmares usually begin. DigiCore handles it natively:

## Under the Hood

For the architects and CTOs reading this section first — here's what powers the platform:

**Backend:** Java 21 on Spring Boot 4.0.4. Clean package architecture with 150+ distinct domain modules following BIAN (Banking Industry Architecture Network) service domain modeling. PostgreSQL 16 with Flyway-managed schema migrations (80 versioned scripts, ~240 tables). Redis 7 for caching with Caffeine as an in-memory fallback. HikariCP connection pooling. Hibernate 6 with JPA for persistence.

**Frontend:** React 18 with TypeScript 5.4. Vite 5.2 for builds. 40+ feature modules, 100+ pages, and dedicated API service files with TanStack Query v5 for server state management. Zustand for client state. React Hook Form + Zod for form validation. Tailwind CSS 3.4, Radix UI for accessible primitives, Recharts for data visualization, and TanStack Table for data-heavy grids.

**Security:** Keycloak OIDC with PKCE authorization code flow. JWT (RS256) validation on every API call. Role-based access control (CBS_ADMIN, CBS_OFFICER, TELLER, BRANCH_MANAGER, COMPLIANCE, TREASURY, LOAN_OFFICER, RISK_OFFICER) plus attribute-based access control (ABAC) for fine-grained policy enforcement. MFA enrollment and challenge. PII masking policies. SIEM correlation rules. Session timeout with recovery.

**API Layer:** RESTful APIs under `/api/v1/` with a consistent `ApiResponse<T>` wrapper, pagination metadata, and centralized exception handling. SpringDoc OpenAPI 3 with Swagger UI. Actuator endpoints for health, metrics, and Prometheus scraping.

**Data Model:** 80 Flyway migrations spanning nine phases — from core customer and account tables through products, cards, payments, regulatory, security, treasury, capital markets, ECL, onboarding, wallets, virtual accounts, and reconciliation. Multi-currency support with configurable deployment profiles (Global/USD, Nigeria/NGN, UK/GBP, UAE/AED, Singapore/SGD).

**Testing:** JUnit 5 with Testcontainers for integration tests against real PostgreSQL and Redis instances. ArchUnit for architecture compliance. REST Assured for API testing. Vitest for frontend unit and integration tests (80% line coverage threshold). Playwright for end-to-end testing with dedicated suites for smoke, accessibility (axe-core), performance, mobile viewport, dark mode, and responsive design.

**Operations:** Gradle 9.1 build system. PM2 process management. Spring Actuator with Micrometer for Prometheus + Grafana monitoring. Environment-specific deployment profiles. MapStruct for compile-time DTO mapping. Lombok for boilerplate reduction.

This isn't a prototype. It's production banking infrastructure.

## Why We Built This

The core banking market has a structural problem.

On one end, you have the legacy incumbents — Temenos, Finacle, Oracle FLEXCUBE. Battle-tested but rigid. On-premise by default. Hostile to modern developer experience. Extending them means COBOL-era abstractions, thick client interfaces, and integration costs that rival the license fee.

On the other end, you have the cloud-native challengers — Thought Machine, Mambu, 10x Banking. Modern architecture, but deliberately narrow. They'll give you a ledger and an API. Maybe deposits. Maybe lending. But treasury? Capital markets? ALM? ECL calculations? Reconciliation? That's "partner ecosystem" — which is enterprise code for "your problem."

DigiCore CBS sits in the space between: **modern architecture with full-spectrum banking coverage.** Every module speaks the language of banking — not fintech abstractions, but real banking operations. Chart of accounts, not wallet balances. Collateral valuations, not JSON metadata. Regulatory returns, not webhook events. Subledger reconciliation, not CSV exports.

We built this because a bank shouldn't need fifteen vendors to run one institution.
